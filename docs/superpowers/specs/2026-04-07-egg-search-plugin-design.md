# 洛克王国世界查蛋器插件设计文档

## 概述

AstrBot 插件，根据精灵蛋的尺寸和重量匹配精灵种类。

## 数据存储

### 文件位置
`{插件目录}/data/egg_data.csv`（相对于插件根目录）

### 数据格式
```csv
精灵名称,尺寸最小,尺寸最大,重量最小,重量最大
精灵A,10.000,15.000,20.000,30.000
```

- 文件编码：UTF-8 with BOM（确保中文兼容）
- 所有数值精确到三位小数
- 用户手动维护此文件

## 指令设计

### 格式
```
/egg 尺寸 重量
```

### 示例
```
/egg 12.5 28.3
```

## 匹配逻辑

1. 解析用户输入的尺寸和重量（支持整数和小数）
2. 从 CSV 文件加载精灵数据
3. 匹配条件：`尺寸最小 <= 输入尺寸 <= 尺寸最大` 且 `重量最小 <= 输入重量 <= 重量最大`
4. 收集所有满足条件的精灵

## 输出格式

### 匹配成功
```
【查询结果】
输入：尺寸 12.500，重量 28.300
匹配精灵：精灵A、精灵B
```

### 无匹配
```
未找到匹配的精灵，请检查输入的尺寸和重量。
```

## 错误处理

### 输入验证
| 情况 | 处理 |
|------|------|
| 参数数量错误 | 提示 "参数格式错误，请使用：/egg 尺寸 重量" |
| 非数字输入 | 提示 "请输入有效的数字" |
| 负数或零值 | 允许，按正常逻辑匹配（由数据本身决定是否匹配） |

### CSV 异常处理
| 情况 | 处理 |
|------|------|
| 文件不存在 | 提示 "数据文件不存在，请联系管理员" |
| 文件为空（仅表头） | 视为无数据，任何查询都返回"未找到" |
| 格式错误（缺列/非法值） | 记录日志，跳过该行继续加载其他数据 |
| 编码错误 | 记录日志，提示 "数据文件编码错误" |

## 数据加载

- 插件初始化时自动加载 CSV 数据到内存
- 提供 `/egg_reload` 指令重新加载数据（方便更新数据后无需重启）
- 匹配结果按精灵名称排序输出

## 技术实现

- 插件初始化时加载 CSV 数据到内存
- 使用 Python 标准 `csv` 模块解析
- 数值比较使用 `float` 类型
